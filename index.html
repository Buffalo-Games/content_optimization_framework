New Item — 6-Step Content Workflow
New Item
1
Gather product inputs
Collect everything the AI needs before opening a prompt window. Missing inputs produce generic output.
  • Product name, SKU, UPC, item dimensions, weight
  • Piece count, materials, age range, player count
  • Key feature differentiators (die cut, foil, licensed art, etc.)
  • Target audience and primary occasion / use case
  • 3 to 5 seed keywords from the Term Bank for this market
2
Generate Amazon Title + Bullets with AI
Paste your inputs into the BHC AI prompt template. Review AI output against the 12 dimensions.
  • Title: max 200 characters, lead keyword in first 60 chars, include brand name and core feature
  • Bullet 1: lead with the primary benefit, not a feature
  • Bullets 2 through 5: one idea per bullet, max 500 chars each, mix features and audience signals
  • Validate format in the Field Validator tab before moving on
3
Write Long Description + Backend Keywords
  • Long description: max 2000 characters, HTML-safe, no repeated bullet content
  • Backend keywords: max 250 bytes total, no punctuation, no brand names, no repetition of title terms
  • Use the Term Bank to find secondary and long-tail keywords for the backend field
4
Run pre-flight validation
Go to the Field Validator tab. Paste each field. All five checks must be green before proceeding.
  • Title max 200 chars
  • Each bullet max 500 chars
  • Long description max 2000 chars
  • Backend keywords max 250 bytes
  • No bullets starting with special characters
5
Enter in Syndigo / PIM
Enter finalized content into Syndigo. Map Syndigo attributes using the Syndigo attribute terms in the Term Bank. Confirm all required fields are populated for each retailer channel.
6
Content QA and publish
Final review before publishing:
  • Read title aloud — does it make sense? Is it grammatical?
  • Confirm Bullet 1 leads with a benefit, not a spec
  • Check that no competitor brand names appear in any field
  • Confirm all images are assigned and in correct sequence
  • Submit for publish in Syndigo once all fields are confirmed green

Digital / eContent Update
Digital
What a digital update covers
  • Updating Syndigo attributes (Used For, Is A, Capable Of, etc.) without touching the physical listing
  • Refreshing A+ content, enhanced images, or video assets
  • Correcting content already live in a retailer portal via Syndigo sync
  • Updating the product description for a retailer-specific feed
What stays the same vs. physical
  • Same: Title, bullets, backend keywords — these are authored once in Amazon Seller Central and flow from there
  • Different: Syndigo attribute fields, enhanced content, and retailer-specific description blocks are managed in Syndigo and can be updated independently
  • Key rule: Never update Amazon title or bullets in Syndigo — Syndigo does not write back to Amazon title or bullet fields. Amazon-side content must be changed in Seller Central.
Syndigo attribute checklist
  • Used For Audience — who is this product for?
  • Used For Event — what occasions or events is it used at?
  • Used For Function — what does it do functionally?
  • Used In Location — where is it used?
  • Capable Of — what outcome does it enable?
  • Is A — what category or type is it? (supports search discovery)
  • Use the Syndigo filter in the Term Bank tab for ready-to-use attribute values

Amazon-First Content Workflow

The BHC content workflow is Amazon-first by design. Amazon has the most demanding character limits and the highest commercial intent traffic. Content authored to Amazon standards is high-quality enough to adapt downstream.

1
Gather product inputs Pre-work
Collect all product details before prompting: name, SKU, dimensions, piece count, materials, age range, player count, key differentiators, and 3 to 5 seed keywords.
  • Missing inputs produce generic AI output -- garbage in, garbage out
  • Use the Term Bank to select seed keywords for your product's market
  • If the item is an extension, note what changed vs. the parent
2
Draft with AI using the BHC framework Amazon
Paste inputs into the 12-dimension BHC prompt template. Generate title, 5 bullets, long description, and backend keywords in one pass.
  • Prompt for Amazon format specifically -- character limits, benefit-first bullets
  • Regenerate individual fields that don't land -- don't accept a full redo
  • The 12 dimensions: Who, What, Where, When, Why, How, Benefit, Feature, Occasion, Trust, Comparison, Brand
3
Validate all fields Quality gate
Run every field through the Field Validator tab. All preflight checks must be green before content moves forward.
  • Title max 200 chars, lead keyword front-loaded
  • Bullets max 500 chars each, no special character at start
  • Long description max 2000 chars, no HTML
  • Backend keywords max 250 bytes, no commas, no repetition
4
Enter validated content into Syndigo PIM
Paste finalized Amazon content into Syndigo fields. Map Syndigo-specific attributes (Used For, Is A, Capable Of, etc.) using the Syndigo term bank.
  • Syndigo manages retailer syndication to Target, Walmart, Kroger, and others
  • Syndigo does NOT write back to Amazon title or bullet fields
  • Amazon Seller Central is always the source of truth for title and bullets
  • Upload images in correct sequence -- hero shot first, lifestyle second
5
Publish and QA live listing Launch
Submit for publish in Syndigo. Verify live on Amazon ~24 to 48 hours after submission. Spot-check character rendering, image load, and bullet formatting.
  • Amazon may override certain fields (title suppression, image policy) -- check live vs. submitted
  • Backend keywords are not visible to shoppers -- verify via Seller Central, not the live listing
  • Flag any suppressed or rejected content in the content tracker
6
Retailer adaptation Future state
Adapt Amazon content for Walmart, Target, and other retailer-specific format requirements.
Why this is not automated yet: Amazon content cannot auto-syndicate to other retailers because each retailer has different character limits, field structures, and content policies. Walmart allows HTML in descriptions; Target has different image requirements; Kroger uses a different attribute taxonomy entirely. Until a retailer-specific templating layer is built, adaptation is a manual step. The Syndigo attribute fields (Used For, Is A, etc.) do syndicate -- those are already live. The title, bullet, and description layer requires retailer-specific review.
